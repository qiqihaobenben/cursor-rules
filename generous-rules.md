# 代码风格和最佳实践
## JavaScript/TypeScript
- 使用 ES6+ 语法特性
- 优先使用 const，其次 let，避免 var
- 使用箭头函数，除非需要 this 绑定
- 使用模板字符串而不是字符串拼接
- 遵循驼峰命名法
- 添加适当的类型注解（TypeScript）

## Vue.js
- 优先使用 Composition API（Vue 3）
- 使用 <script setup> 语法糖
- 组件名使用 PascalCase
- 合理使用 ref 和 reactive
- 遵循 Vue 3 最佳实践

## React
- 优先使用函数组件和 Hooks
- 使用 JSX 语法
- 组件名使用 PascalCase
- 合理使用 useState, useEffect 等 Hooks
- 避免不必要的重渲染

## Java
- 遵循 Java 命名约定
- 使用合适的设计模式
- 添加适当的异常处理
- 使用 Spring Boot 最佳实践
- 合理使用注解

## 数据库 (MySQL)
- 表名使用下划线命名法
- 字段名使用下划线命名法
- 添加适当的索引建议
- 考虑查询性能优化
- 提供数据库设计建议
